When does deep learning actually help on prompt-injection detection?

Three models, eight tool classes, one operating point. A keyword heuristic collapses from 68% to 5% recall when benigns contain natural imperative phrasing. A 66M-parameter DistilBERT beats XGBoost by ~4 points — but only on natural-language tools (email, code, messaging). On structured-JSON tools (calendar, shopping), classical matches deep. Tool-identity conditioning is a null result across both models.

Method in one paragraph

The dataset is built from InjecAgent's 1,054 attack cases across 17 user tools that map to 8 tool classes. For each attack we emit a matched (benign, malicious) pair anchored on the same Tool Response Template shell — only the <Attacker Instruction> slot differs. Benign fillers come from a mixed-register pool of 40 entries: half conversational declaratives, half benign-intent imperatives. The imperative half is what forces the classifier to learn malicious intent rather than imperative shape — otherwise char n-grams hit 100% trivially on every fold. Three models are trained: a keyword heuristic, XGBoost over char n-gram TF-IDF + tool one-hot, and a fine-tuned DistilBERT with [TOOL: <class>] prepend. The primary metric is recall at 1% FPR, calibrated on val and applied to the held-out class.
